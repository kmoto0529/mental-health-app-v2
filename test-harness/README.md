# Aside API接続 検証ハーネス

AIコネクションエラーの原因特定と対策検証のためのNodeスクリプト。
アプリと**同一のリクエスト構造**でGemini APIを叩き、エラー発生率・リトライ回数・失敗モードを集計する。

## 使い方

1. プロジェクトルートに `.env` を作成:
   ```
   GEMINI_API_KEY=AIza...（アプリで使っているAPIキー）
   ```

2. 実行:
   ```bash
   node test-harness/run.js
   ```

3. 結果は `test-results/YYYYMMDD_HHMMSS.json` に保存される

## 測定項目

- 各呼び出しのステータス（200 / 429 / 503 / etc）
- リトライ回数
- 成功までの経過時間
- 最終的な成功/失敗
- エラー詳細ログ

## シナリオ

1. **chat_initial**: チャット初回挨拶（短い入力→短い出力）
2. **chat_rally**: 会話ラリー（長い履歴→通常出力）
3. **alt_first**: もやもや整理の初回代替案生成
4. **alt_regen**: もやもや整理の再生成（+他の見方）

各シナリオを複数回実行して統計を取る。
